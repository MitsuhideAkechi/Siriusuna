# 抽象
物理 -> 器件 -> 模拟电路 -> 数字电路 -> 逻辑 -> 微体系结构 -> 体系结构 -> 操作系统 -> 应用软件

**本书集中于数字电路到体系结构之间的抽象**

# 约束
我的理解是，标准化、模块化，提供接口，用以服务更高层次的抽象。

# 3Y原则
- 层次化：将系统按 **功能、逻辑或抽象程度** 划分为不同层次，每层承担明确职责，层间通过清晰接口交互。
- 模块化：将系统拆解为 **独立、可复用的模块**，每个模块聚焦单一功能，模块内 “高内聚”（功能紧密关联），模块间 “低耦合”（依赖弱）。
- 规整化：强调系统的 **结构、接口、实现规范统一**，追求 “整齐、可预测”。
